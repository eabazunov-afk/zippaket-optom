# Дизайн: Редизайн сайта + интернет-магазин ZLOCK (zip-пакеты оптом)

**Дата:** 2026-06-18
**Статус:** на ревью
**Автор:** brainstorming-сессия

## Проблема и цель

Сайт `zippaket-optom.ru` — лендинг производителя zip-пакетов с лид-генерацией
(заявки → amoCRM/Telegram, калькулятор). Цель переработки:

1. **Конверсия** — больше заявок/продаж.
2. **Premium-восприятие** — современный, дорогой, технологичный вид.
3. **Удобство (UX)** — понятный путь от выбора товара до покупки.

Главное расширение: добавить **полноценный интернет-магазин** — каталог с ценами →
корзина → оформление заказа → онлайн-оплата.

## Решения (зафиксированы в брейншторме)

| Вопрос | Решение |
|---|---|
| Стек | Остаёмся на PHP, редизайн и магазин «на месте» (Вариант A — расширяем текущий код) |
| Объём редизайна | Все публичные страницы (главная, каталог, товар, «с логотипом», политика). Админку не трогаем визуально |
| Брендинг | Сохраняем логотип ZLOCK и название; цвета/шрифты/компоновка — свобода |
| Визуальное направление | C — Современный Premium: тёмный градиент, стекло, мятный акцент, скругления |
| Модель магазина | Классический: каталог → корзина → оплата. Калькулятор остаётся как инструмент |
| Платёжный провайдер | ЮKassa (за абстракцией `PaymentGateway`, провайдер сменяем) |
| Аккаунты | Гостевой заказ, без регистрации |
| Доставка | Простые варианты (самовывоз/курьер/ТК по РФ); стоимость ТК менеджер согласует после заказа |

## Цели (Goals)

- Каталог товаров на основе существующей таблицы `products` (~49 позиций с ценами/остатками).
- Корзина (в сессии), оформление заказа, онлайн-оплата ЮKassa.
- Поддержка физлиц и юрлиц (ИНН/КПП/счёт), выбор способа оплаты (онлайн/по счёту).
- Минимальное количество и кратность упаковке у товаров.
- Premium-редизайн всех публичных страниц.
- Каждый оплаченный заказ → amoCRM + Telegram + email + админка.

## Не-цели (Non-Goals)

- Личные кабинеты покупателей (только гость).
- Автоматический расчёт стоимости доставки через API ТК (можно добавить позже).
- Редизайн админки.
- Полный переезд на новый стек / CMS.
- Удаление существующей лид-модели и калькулятора.

## Визуальная система

- **Палитра:** фон `#0B1F3A` / `#12345C`; акцент мята `#5FE3D0` / `#0A8F8F`;
  CTA/акция `#FFB020`; светлый фон контента `#F4F7FB`.
- **Типографика:** Inter (800 заголовки / 600 подзаголовки / 400 текст).
- **Компоненты:** карточки 16px скругление + мягкие тени; кнопки 30px;
  тёмные «стеклянные» блоки на градиенте, светлые карточки для контента/товаров.

## Архитектура (Вариант A — расширяем текущий PHP)

Процедурный PHP + PDO, в стиле существующего кода. Новые модули:

```
www/
  catalog.php            — витрина каталога (фильтры: категория/размер/цвет, сетка карточек)
  product.php            — страница товара (?id=)
  cart.php               — корзина
  checkout.php           — оформление заказа
  order_success.php      — статус оплаты / «спасибо»
  includes/
    cart.php             — логика корзины (сессия)
    order.php            — создание/чтение заказов
    payment/
      PaymentGateway.php   — интерфейс (абстракция шлюза)
      YooKassaGateway.php  — реализация ЮKassa
    payment_callback.php   — webhook ЮKassa (смена статуса заказа)
  admin/orders.php         — заказы в админке (+ смена статусов, подтверждение оплаты по счёту)
```

Переиспользуем: таблицу `products`, шапку/футер, интеграции amoCRM/Telegram, reCAPTCHA.

## Модель данных

**Изменения в `products` (новые поля):**
```
min_order_qty   — минимальное количество к заказу
qty_step        — кратность / размер упаковки
pack_label      — подпись упаковки (напр. «уп. 50 шт»)
```

**Новая таблица `orders`:**
```
id, order_number, status,
customer_type      — individual | company
customer_name, phone, email,
company_name, inn, kpp, legal_address, needs_invoice,   -- для юрлиц
delivery_method    — pickup | courier | tk
delivery_address, comment,
payment_method     — online | invoice
items_total, total,
payment_id, payment_status,
created_at, updated_at
```

**Новая таблица `order_items`:**
```
id, order_id, product_id,
name_snapshot, price_snapshot,   -- копия на момент заказа
qty, line_total
```
Цена и название копируются в `order_items` (snapshot) — изменение прайса позже
не искажает старые заказы.

**Статусы заказа:** `new` → `pending_payment` → `paid` → `processing` → `shipped`
→ `done`; либо `canceled`. Платёжный статус синхронизирует webhook ЮKassa.

## Страницы и путь покупки

Страницы: Главная (лендинг + вход в каталог) · Каталог · Товар · «Пакет с логотипом»
(есть) · Калькулятор (на главной) · Корзина · Checkout · Успех · Политика (есть) ·
Админка с заказами.

Путь: Каталог/Товар → Корзина → Checkout (контакты, физ/юрлицо, доставка, способ
оплаты) → создаётся заказ → **онлайн (ЮKassa)** или **по счёту (юрлицо)** →
webhook/подтверждение → «оплачено» → автоуведомление в amoCRM + Telegram + email + админку.

## Структура главной (порядок блоков)

10. SEO-блок (низ)

## Каталог и карточка товара

- **Каталог:** панель фильтров (категория/размер/цвет) + сетка карточек
  (фото, название, цена, «В корзину»).
- **Товар:** крупное фото, размеры/цвет тегами, цена за единицу, остаток
  (`stock_quantity`), счётчик количества (старт с `min_order_qty`, шаг `qty_step`,
  ручной ввод округляется вверх до кратного), «В корзину», связка с калькулятором
  («Нужен с логотипом? Рассчитать»). Описание/характеристики/доставка — вкладками.

## Корзина и checkout

- **Корзина:** позиции с подписью «мин. N · кратно M», счётчик, сумма по строке,
  итог; пометка «доставку менеджер согласует после заказа».
- **Checkout:** переключатель Физлицо/Юрлицо (для юрлица доп. поля
  company_name/inn/kpp/legal_address/needs_invoice); выбор доставки
  (самовывоз/курьер/ТК); выбор оплаты (онлайн / по счёту — только юрлицам);
  согласие с политикой + reCAPTCHA.

## Оплата

- ЮKassa за интерфейсом `PaymentGateway` (провайдер сменяем).
- Заказ `new` → `pending_payment`, создаётся платёж → редирект на форму ЮKassa
  (карта/СБП).
- **Webhook `payment_callback.php`:** проверка подлинности источника, идемпотентная
  смена статуса. Источник истины о сумме — серверный (заказ из БД), не запрос клиента.
- Возврат на `order_success.php`. Чек по 54-ФЗ формирует ЮKassa (передаём состав заказа).
- **По счёту (юрлица):** заказ `pending_payment`, генерация счёта (номер/PDF),
  менеджер подтверждает оплату вручную в админке → `paid`.

## Безопасность

- Вынести секреты из `config.php` (пароль БД, токены amoCRM, ключ reCAPTCHA) в
  окружение/`.envrc`; убрать из-под git (`.gitignore` готов).
- Все запросы к БД — подготовленные выражения (PDO).
- CSRF-токены на формах корзины/checkout; reCAPTCHA на оформлении.
- Серверная валидация цен, количеств (мин/кратность), сумм — клиенту не доверяем.
- Webhook: проверка подлинности + идемпотентность.

## SEO

- Динамический `sitemap.xml` (категории + товары).
- ЧПУ для каталога/товаров; `meta_title`/`meta_description` из `products`.
- JSON-LD `Product` + `Offer` (цена, наличие); `BreadcrumbList` (хлебные крошки).

## Сопутствующие исправления (из research)

- Баг `SITE_URL` = `'https://https://zippaket-optom.ru/'` (двойной `https://`).
- Просроченный токен amoCRM (истёк 2026-05-31) — перевыпуск/refresh, иначе заказы
  и заявки не уходят в CRM.

## Обработка ошибок

- Недоступность ЮKassa → понятное сообщение, заказ сохранён (оплата позже).
- Пустая корзина; товар стал неактивен/закончился на этапе оформления.
- Некорректное количество (не кратно/меньше минимума) — отклонение на сервере.

## Тестирование

- **Юнит:** расчёт сумм корзины; валидация мин/кратности; статусная машина заказа.
- **Интеграция:** webhook (оплачено/отменено/повтор); создание заказа.
- **Платежи:** тестовый магазин ЮKassa (sandbox).
