# Решение: редизайн в продающую B2B-витрину ZIP-пакетов

> Синтез 4 агентов (маркетолог-стратег + фронт/UX + бэкенд/данные + SEO/перф).
> Данные 2026. Deliverable — стратегия + план, БЕЗ кода. Статус: НА УТВЕРЖДЕНИЕ.

## 0. Главный вывод

Полноценная «стройка с нуля» не нужна. Три тех-аудита независимо сошлись: сайт готов
к витрине на **70–85%**. Тёмная дизайн-система цельная (`--z-*` токены), 80% фронта
переиспользуется, а бэкенд уже несёт то, что обычно строят с нуля:

- **есть**: калькулятор опт-цены (`calculator.php`), опт-градация скидок, MOQ/шаг,
  остатки/наличие, фильтр-подбор (размер/толщина/замок/цвет), хиты (`getPopularProducts`),
  таблицы `leads` + `visits` + `calculations`, UTM-трекинг, amoCRM, YooKassa (54-ФЗ),
  Telegram/email-уведомления, sitemap.php, ЧПУ (.htaccess), базовый Schema.org.
- **нет / заглушки**: отзывы (хардкод в `index.php:394`), соц-доказательства (лого/кейсы),
  FAQ, лид-магнит «скачать прайс», кнопка «Запросить КП» (RFQ), новинки
  (`created_at` есть, метода `getNewProducts()` нет), реальные фото товаров (плейсхолдеры).

Значит задача = **пересобрать главную в продающий порядок + закрыть 6–8 маркетинговых
дыр + SEO/перф-харденинг**, а не переписывать движок.

## 1. Концепция витрины (от маркетолога, подтверждена тех-реальностью)

Главная = самообслуживающийся оптовый прайс-конфигуратор. Оптовик за 30 секунд понимает
«что, почём от объёма, как быстро отгрузят» и оставляет заявку/скачивает прайс без звонка.

**Новый порядок секций главной:**
1. Hero: оффер под опт + 3–4 маркера доверия (МОQ, наличие, НДS/док-ты, срок отгрузки) +
   2 CTA («Скачать прайс» / «Запросить КП») + строка поиска-подбора.
2. Поиск/фильтр-подбор (размер / толщина / тип замка) → ведёт в каталог.
3. Хиты продаж (из БД, цена «от» + опт-градация + наличие).
4. Новинки (из БД по `created_at`).
5. Калькулятор оптовой цены (уже есть — вынести выше).
6. Блок доверия: лого клиентов-юрлиц, отзывы компаний, кейсы отгрузок.
7. Условия опта: МОQ, скидки по объёму, доставка, документы/НДС.
8. Как работаем (шаги от заявки до отгрузки).
9. FAQ по опту (SEO + снятие возражений, FAQPage schema).
10. Финальный CTA «Получить прайс/КП» + контакты + мессенджеры.

Каталог: слева фильтры-подбор; карточка — цена «от», МОQ, наличие, «В корзину» +
«Быстрый заказ в 1 клик» + «Запросить КП».

## 2. Реконсиляция: фишка → приоритет → тех-реальность → вердикт

| Фишка (маркетолог) | Прио | Тех-реальность (аудит) | Вердикт / усилие |
|---|---|---|---|
| Калькулятор опт-цены | P0 | ЕСТЬ `calculator.php` + модалка | Переиспользовать, поднять выше · **S** |
| «Запросить КП» (RFQ) | P0 | Есть `leads`+amoCRM+lead-форма | Новая кнопка → существующий лид-флоу · **S** |
| Градация скидок по объёму | P0 | Есть, но множители хардкод (`z_price` 0.82/0.92) | Параметризовать в config · **S** |
| MOQ на карточке | P0 | ЕСТЬ (`min_order_qty`, показывается) | Готово · **—** |
| Остатки/наличие + срок | P0 | ЕСТЬ (`stock_quantity`, прогресс-бары) | Добавить «срок отгрузки» текстом · **S** |
| Фильтр-подбор | P0 | ЕСТЬ полная система фильтров | Готово, вынести в hero · **S** |
| Скачать прайс (лид-магнит) | P0 | Нет генерации; `leads` для захвата есть | Генерация PDF/XLS из БД + захват · **M** |

| Хиты продаж | P0 | ЕСТЬ `getPopularProducts()` | Готово · **—** |
| Новинки | P1 | `created_at` есть, метода нет | Добавить `getNewProducts()` · **S** |
| Отзывы юрлиц | P0 | ЗАГЛУШКИ (хардкод) | Нужна таблица `reviews` + админ-CRUD · **M** |
| Кейсы отгрузок / лого клиентов | P0 | Нет | Таблица `testimonials` ИЛИ статика + контент владельца · **M** |
| Гарантии/документы (НДС, ГОСТ) | P1 | Мини-блок есть | Расширить · **S** |
| Schema Product/Offer + Rating | P1 | Частично (Product неполный) | Дополнить stock/rating/url · **M** |
| FAQ по опту + FAQPage schema | P1 | Нет | Статика или таблица `faq` · **S** |
| Мессенджеры/обратный звонок | P1 | Telegram-ссылки есть | Плавающий виджет · **S** |
| Мобильная скорость <3с (CWV) | P1 | Нет WebP/srcset, font-display, 185KB CSS | Перф-трек (см. §4) · **M** |
| Брошенная корзина-догон | P1 | Сессия есть, лог/уведом нет | Таблица `abandoned_carts` + notify · **M** |
| Сравнение товаров | P1 | Нет | localStorage-компонент · **M** |
| Личный кабинет (реордер, свои цены) | P1 | Нет | Отдельная крупная задача · **L (отложить)** |
| Подписка на прайс (email) | P2 | Нет таблицы | Таблица `subscribers` · **S–M** |
| Образцы / пробная мини-партия | P1 | Есть CTA «образец» на главной | Довести до формы-заявки · **S** |
| Персональные цены залогиненным | P2 | Нет | После кабинета · **L (отложить)** |

**Уточнения из 2026-ресёрча (маркетолог, 2-й проход):**
- **Открытые опт-цены «от N ₽/шт» в карточке — фишка №1 по ROI** (73% B2B готовы
  заказывать онлайн, если видят цену). Это фундамент, а не опция.
- **Брошенная корзина**: инсентив (скидка/образец) — только во 2-м–3-м письме, не в 1-м
  (иначе приучаем бросать корзину). Мерить **CTR и заказы, НЕ open rate** (Apple MPP искажает).
- **SEO категорий**: не «тонкая» выгрузка фида (Google 2026 понижает), а уникальный
  buyer-guide на категорию под интент.

## 3. Дорожная карта (фазы)

### Фаза A — P0, «монетизация трафика» (быстрый эффект, почти всё reuse)
- A1. Пересборка главной в новый порядок секций (§1).
- A2. Hero: оффер под опт + 2 CTA + строка поиска-подбора.
- A3. Секции «Хиты» и «Новинки» из БД (`getNewProducts()` — новый метод).
- A4. Параметризовать опт-множители скидок (убрать хардкод в config).
- A5. Кнопка «Запросить КП» (RFQ) на карточках/hero → существующий лид-флоу + amoCRM.
- A6. «Быстрый заказ в 1 клик» (тел. + позиция) → лид.
- A7. Лид-магнит «Скачать прайс» (PDF/XLS из БД, захват email/тел.).

### Фаза B — P0/P1, «доверие и юзабилити»
- B1. Отзывы: таблица `reviews` + админ-CRUD + вывод на главной и карточке (заменить заглушки).
- B2. Соц-доказательства: лого клиентов + кейсы отгрузок (таблица `testimonials` или статика).
- B3. FAQ по опту + FAQPage schema.
- B4. Блок гарантий/документов (НДС, ГОСТ/сертификаты, договор).
- B5. Плавающий виджет мессенджеров/обратного звонка.

### Фаза C — SEO/перф-харденинг (см. §4)
### Фаза D — P1/P2, удержание (отдельные итерации)
- D1. Брошенная корзина-догон. D2. Сравнение. D3. Подписка на прайс.
- D4. Личный кабинет + персональные цены (крупно, отдельный план).

## 4. SEO/перф-трек (Фаза C)

**Критично (2026):**
1. `canonical` на главной + всех статических/юр-страницах.
2. Уникальные `meta description` + Open Graph на статических страницах (`page_head.php`).
3. Product JSON-LD: добавить `url`, `availability` (InStock), stock level, `aggregateRating`.
4. JSON-LD Product в карточках листинга каталога.
5. `<meta name="robots" content="index, follow">` на индексируемых.

**Высокий приоритет:**
6. WebP/AVIF + `srcset`/`<picture>` для hero и фото товаров (папка images 42 МБ).
7. `&display=swap` к Google Fonts (сейчас риск FOIT → бьёт LCP).
8. Минификация/объединение CSS (185 КБ, легаси `style.css` 78 КБ) + `defer` на JS.
9. Расширить Organization → LocalBusiness.
10. Починить противоречие в `robots.txt` (строки 6–8: Disallow `/*.php$` vs Allow).

**Контент:** текстовые SEO-блоки на главной; поле `description` товара на `product.php`.

## 5. Новые таблицы БД (сводно)

`reviews`, `testimonials` (лого/кейсы), `faq`, `subscribers`, `abandoned_carts`.
Опционально позже: `coupons`, `flash_sales`, `wishlist`, `product_images` (галерея).
Всё остальное собирается из существующих полей без новых таблиц.

## 6. Антипаттерны — НЕ делать (убьёт B2B-доверие)

Фейковые таймеры со сбросом; накрученные «осталось 2 шт / N смотрят» (для опта абсурд);
агрессивные поп-апы/nagging; «цена только по звонку» без ориентира; принудительная
регистрация до КП; фейковые отзывы/лого без согласия.

## 7. Метрики успеха

CR в заявку/КП/заказ (цель — обгон бенчмарка опта ~2,4–2,6%); число «Запросить КП» и
«Быстрый заказ»/нед; использование калькулятора → заявка; скачивания прайса и конверсия
лида; средний чек/объём партии; CR страниц с кейсами/отзывами; recovery брошенных;
органика по «…оптом», покрытие rich-сниппетами, мобильный LCP <2.5с.

## 8. Решения владельца (ЗАФИКСИРОВАНО)

1. **Отзывы — ОБА варианта, поэтапно.** Сначала кураторские кейсы/лого (быстро, в Фазе B),
   затем таблица `reviews` + админ-CRUD с модерацией. Нужен контент от владельца:
   тексты кейсов + логотипы клиентов с согласием.
2. **Фото — ПОКА плейсхолдеры.** Витрину собираем на текущих заглушках (eva/pvd/gripper).
   Галерея товаров и WebP/srcset для реальных фото — откладываются до появления фото.
   В Фазе C делаем WebP/srcset только для hero и существующих изображений.
3. **Прайс-лид-магнит — АВТО-ГЕНЕРАЦИЯ из БД.** PDF/XLS строится из актуальных цен/остатков
   (Фаза A7). Всегда актуален, не требует ручного обновления файла.
4. **Личный кабинет — ОТЛОЖИТЬ.** Остаётся в Фазе D как отдельная итерация. Персональные
   цены залогиненным (P2) — после кабинета.

## 9. Итоговый объём первой волны (A + B + C)

Фаза A (P0, монетизация) → Фаза B (доверие: кейсы/лого сейчас, `reviews` следом) →
Фаза C (SEO/перф-харденинг). Фаза D (брошенная корзина, сравнение, подписка, кабинет) —
отдельными итерациями после запуска первой волны. Фото и галерея — при поступлении
реальных снимков.
