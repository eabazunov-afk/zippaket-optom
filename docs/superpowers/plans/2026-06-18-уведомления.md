# Уведомления о заказе — Implementation Plan

**Goal:** При оформлении заказа и при его оплате оповещать команду и клиента по нескольким каналам: Telegram (админу), Email (админу и клиенту), лид в amoCRM. Переиспользуем существующие интеграции (`includes/amocrm.php::sendToAmoCRM`, `functions.php::sendEmail`, Telegram-бот `tg/`).

**Architecture:** Ядро — **чистые форматтеры** (`includes/notify/order_message.php`): из строки заказа и позиций собирают текст Telegram, тему/тело письма и payload для amoCRM. Без сети и БД → покрыты юнит-тестами. **Каналы** изолированы: `telegram_notify.php` (отправка с инъекцией HTTP-клиента), email через `sendEmail()`, amoCRM через `sendToAmoCRM()`. **Оркестратор** (`order_notifier.php`) дёргает каналы по принципу **best-effort**: каждый канал в своём try/catch, сбой логируется и НИКОГДА не ломает оформление заказа или обработку webhook.

**Tech Stack:** PHP 8.3.31, PDO/MySQL 8.0, PHPUnit 9.6, cURL. Telegram Bot API, amoCRM API v4, `mail()`.

## Global Constraints

- Уведомления — побочный эффект, не критичный путь: ошибки каналов не влияют на статус заказа.
- Форматирование — чистые функции с тестами; отправка — тонкие обёртки.
- Идемпотентность «оплачен»: уведомление шлём только при ПЕРВОМ переходе заказа в `paid`
  (`order_apply_payment_status` возвращает `applied`), на повторах webhook — `noop`, не шлём.
- Секреты каналов — в своих файлах (`tg/config.php`, `config.php`), не в git.
- Telegram-бот `tg/bot.php` НЕ подключаем (у него сайд-эффекты webhook'а) — свой минимальный отправитель.

## File Structure

- `www/includes/notify/order_message.php` — ЧИСТЫЕ форматтеры: `format_order_telegram`,
  `format_order_email` (subject/body), `order_to_amocrm_data`, хелперы `notify_money`/`notify_label`.
- `www/includes/notify/telegram_notify.php` — `telegram_send` (инъекция HTTP), `telegram_configured`.
- `www/includes/notify/order_notifier.php` — оркестратор: `notify_new_order`, `notify_order_paid`,
  обёртка `notify_channel` (best-effort).
- `www/includes/order.php` — добавлены `order_items_get`; уточнён возврат `order_apply_payment_status`
  (`applied` только при реальном переходе, иначе `noop`).
- `www/checkout.php` — после создания заказа → `notify_new_order`.
- `www/api/payment_callback.php` — при первом переходе в `paid` → `notify_order_paid`.
- `www/tests/OrderMessageTest.php`, `www/tests/TelegramNotifyTest.php` — тесты.

---

## Flow

1. Оформление заказа (`order_create` ok) → `notify_new_order(order, items)`:
   Telegram(админ) + Email(админ) + лид amoCRM. Затем — обычный флоу (оплата/успех).
2. Webhook оплаты, первый переход `pending_payment → paid` (`applied`) →
   `notify_order_paid(order, items)`: Telegram(админ) + Email(админ) + Email(клиент, если есть).
